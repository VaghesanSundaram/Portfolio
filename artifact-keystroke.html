Keystroke Dynamics Authentication

A security project that uses behavioral biometrics, specifically typing patterns, as an additional signal for user authentication.

Context and purpose

This project implements a keystroke dynamics based authentication system that collects timing features such as key hold times and flight times between keystrokes. Using these features, I trained models that distinguish between legitimate users and impostors, experimenting with thresholds and classification approaches. The work grew out of my interest in authentication mechanisms that do not rely solely on passwords, and it connects directly to my broader focus on applied security and user protection.

I designed the project to be reproducible and transparent. That meant carefully structuring the data pipeline, documenting feature extraction, and providing clear evaluation metrics so that others could understand not only the model results, but also the limitations and possible attack surfaces. As a result, the artifact functions both as an implementation and as a technical explanation of behavioral biometric authentication.

What this artifact reveals

This project demonstrates my curiosity about nontraditional authentication factors and my willingness to bridge theory and practice. I had to understand the research literature on keystroke dynamics while also making pragmatic decisions about data collection, model selection, and usability tradeoffs. It reveals how I think about security not just as cryptography, but as a human centered problem where false positives, false negatives, and user experience all matter.

The keystroke dynamics project also highlights my growth as a communicator. I needed to explain to peers why timing data can be sensitive, how classification thresholds influence risk, and why behavioral biometrics should be layered with other factors rather than used on their own. This kind of explanation practice will be critical in future roles where I need to articulate security risks and mitigations to both technical and nontechnical stakeholders.
